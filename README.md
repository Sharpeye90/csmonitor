# CS2 Match Parser

Веб-приложение для загрузки итогового скриншота CS2, распознавания статистики матча и сохранения данных в PostgreSQL.

## Что умеет

- принимает JPG/PNG/WebP со счетом итогового табло CS2;
- определяет дату матча от времени загрузки;
- если загрузка была до 06:00 по `APP_TIMEZONE`, записывает матч предыдущей датой;
- извлекает карту, счет, игроков по командам, убийства, смерти, урон и `%ГЛ`;
- формирует `KDA` в формате `Убийства/Смерти`;
- сохраняет матч, команды и игроков в PostgreSQL;
- показывает последние 10 матчей на главной странице.

## Стек

- `Next.js 15`
- `Prisma`
- `PostgreSQL`
- `OpenAI Responses API` для распознавания скриншота

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Скопировать `.env.example` в `.env` и заполнить:

```bash
cp .env.example .env
```

3. Применить миграции:

```bash
npx prisma migrate dev --name init
```

4. Запустить проект:

```bash
npm run dev
```

## Облачный деплой

Проще всего развернуть так:

1. `Vercel + Neon/Supabase Postgres`
2. `Railway` целиком
3. `Fly.io` через `Dockerfile`

Минимальные переменные окружения:

- `DATABASE_URL`
- `OPENAI_API_KEY`
- `OPENAI_MODEL`
- `APP_TIMEZONE`

После деплоя нужно выполнить миграции Prisma:

```bash
npx prisma migrate deploy
```

## Структура БД

- `Match` хранит дату матча, карту, счет и время загрузки.
- `Team` хранит сторону и итоговый счет команды.
- `PlayerStat` хранит игрока и его показатели.

## Важное замечание

Точность распознавания зависит от качества скриншота. Для production стоит добавить:

- ручную валидацию результата перед сохранением;
- дедупликацию матчей;
- журнал ошибок OCR;
- повторную обработку неуверенных распознаваний.
